# Evidence Linkage Model

## Purpose

Defines how scope verification is linked to evidence through inspection steps, commissioning steps, and closeout requirements. Evidence is the proof that scope has been correctly executed.

## Evidence Categories

### 1. Inspection Evidence
Evidence generated by quality inspection steps during construction:
- Pre-cover photographs documenting substrate conditions
- Adhesion test results for membranes and coatings
- Air leakage test reports
- Flood test documentation
- Visual inspection reports with deficiency logs

### 2. Commissioning Evidence
Evidence generated by BECx activities:
- Design review checklists
- Construction observation reports
- Pre-cover inspection documentation
- Performance test results (air, water, thermal)
- Seasonal observation reports

### 3. Closeout Evidence
Evidence collected during project closeout:
- Warranty certificates and NDL letters
- As-built drawings reflecting actual construction
- Maintenance and operations manuals
- Training records
- Spare materials inventory

## Linkage Structure

```
Scope of Work
  |-- Inspection Step --> evidence_required: ["photographs", "test_report"]
  |-- Commissioning Step --> documentation_required: ["observation_report"]
  |-- Closeout Requirement --> (implicit evidence by type)
  |     |-- Warranty Handoff --> documentation_required: ["warranty_letter"]
```

## Evidence Requirements by Inspection Type

| Inspection Type | Typical Evidence Required |
|---|---|
| `pre_cover` | Photographs, substrate report |
| `substrate` | Moisture test results, profile measurements |
| `adhesion` | Pull test results |
| `continuity` | Mapped photographs showing continuous coverage |
| `penetration` | Detail photographs, seal verification |
| `transition` | Detail photographs showing lap and termination |
| `flood_test` | Test report with duration, observations |
| `air_test` | Test report per ASTM E2357 |
| `visual` | Inspection report with deficiency log |

## Evidence Integrity Rules

1. Evidence is referenced by scope records but stored externally. The Scope Kernel does not store evidence files.
2. The `evidence_required` field on inspection steps defines what must be collected.
3. Evidence completeness is a closeout verification requirement.
4. Missing evidence for a hold-point inspection is a blocking condition.

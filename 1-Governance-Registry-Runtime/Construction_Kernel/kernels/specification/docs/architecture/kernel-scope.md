# Kernel Scope — Construction Specification Kernel

## Domain Scope

This kernel covers specification-domain truth for CSI Division 07 — Thermal and Moisture Protection (Building Envelope Systems). The scope encompasses all structured specification content that governs what must be provided, prohibited, tested, warranted, and submitted for Division 07 work.

## Entity Categories

### Specification Sections

CSI MasterFormat sections within Division 07. Each section is a container for requirements, prohibitions, allowances, and associated obligations. Sections are identified by six-digit codes (e.g., 07 52 00 — Modified Bituminous Membrane Roofing) and linked to control layers and interface zones per the shared registries.

### Requirements

Mandatory, recommended, or permissive specification statements. Each requirement carries an obligation level (shall/should/may), a source pointer, and optional linkages to performance criteria, test methods, standards, climate context, and geometry context.

### Prohibitions

Explicit statements of what is not permitted. Prohibitions identify forbidden materials, methods, conditions, or configurations. Each prohibition has a source pointer and may specify the conditions under which it applies.

### Allowances

Explicit statements of what is permitted as an alternative or substitution. Allowances may be conditional (e.g., "permitted only with written approval") and carry source pointers.

### Submittal Requirements

Documents and items that must be submitted for review. Types include product data, shop drawings, samples, test reports, certificates, warranties, and mix designs. Each submittal has timing and review requirements.

### Testing Requirements

Required tests that verify specification compliance. Types include field tests, laboratory tests, mock-ups, and preconstruction testing. Each test has a method reference, acceptance criteria, timing, and frequency.

### Warranty Requirements

Warranty obligations specified for materials, systems, and workmanship. Types include manufacturer standard, manufacturer extended, system warranty, no-dollar-limit (NDL), and workmanship warranties. Each has a duration and conditions.

### Qualification Requirements

Required qualifications for manufacturers, installers, testing agencies, and inspectors. Each qualification specifies minimum experience, required certifications, and status.

### Standards References

Citations to governing standards and codes. Referenced by identifier only (e.g., "ASTM D4541") without reproducing standards text. Linked to the shared_standards_registry.json.

### Source Pointers

Traceable references to originating documents — project manuals, addenda, RFIs, bulletins, and standards body publications. Every specification fact must have at least one source pointer.

### Specification Revisions

Addenda, bulletins, and RFI responses that modify specification content. Each revision creates a lineage chain linking the new record to the superseded record.

## Scope Boundaries

This kernel does not extend to Division 07 work that falls outside specification truth — cost data, scheduling, means and methods beyond what the spec dictates, or trade coordination logistics. Those concerns belong to other systems.

# Reference Intelligence Linkage — Construction Material Kernel

## Relationship to Reference Intelligence

The Construction Material Kernel provides structured material truth to the Construction_Reference_Intelligence layer. The intelligence layer reads material data, correlates it with data from sibling kernels, and produces cross-domain insights. This kernel does not perform intelligence operations.

## Data Flow Direction

```
Material Kernel ──(provides material truth)──> Reference Intelligence
Reference Intelligence ──(provides shared enums, registries)──> Material Kernel
```

Material truth flows outward. Shared vocabularies and registries flow inward. No circular dependencies exist.

## What This Kernel Provides to Intelligence

| Data Type | Intelligence Use |
|---|---|
| Material property records | Cross-reference with specification requirements |
| Compatibility matrices | Identify risk patterns across assemblies |
| Weathering behavior data | Correlate with climate zone analysis |
| Hygrothermal properties | Feed moisture migration risk models |
| Evidence linkages | Validate data quality and currency |
| Ambiguity flags | Surface unresolved material questions |

## What Intelligence Provides to This Kernel

| Artifact | Kernel Use |
|---|---|
| shared_enum_registry.json | Controlled vocabulary for enum fields |
| shared_taxonomy.json | Material class taxonomy alignment |
| shared_standards_registry.json | Standards citation format and IDs |
| control_layers.json | Control layer registry for mappings |
| interface_zones.json | Interface zone registry for context |
| division_07_posture.json | Division 07 domain alignment |

## Linkage Contracts

- Material records reference shared enum values by ID, never by embedding
- Intelligence layer reads material schemas to validate query structure
- Material changes trigger version increments visible to the intelligence layer
- Intelligence layer does not write to material kernel records

## Linkage Boundaries

- Intelligence correlation results are not stored in this kernel
- Risk scores generated by intelligence are not material truth
- Design guidance produced by intelligence references material data but is not owned here
- Pattern analysis across kernels is an intelligence function, not a material function

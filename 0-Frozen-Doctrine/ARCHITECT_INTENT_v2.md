# Construction OS v2 — Architect Intent

## Authority

Armand Lefebvre, L0 — Lefebvre Design Solutions LLC

## Intent Statement

Construction OS v2 is a governed domain operating system for the construction
industry. It is designed to:

1. **Separate truth from execution** — Intent Truth and Actual Truth are compared,
   never collapsed into a single plane
2. **Preserve governance above domain** — ValidKernel governance controls Construction OS,
   not the reverse
3. **Enable governed absorption** — Existing subsystem repos are absorbed into the
   monorepo under strict lineage tracking, not copied or merged casually
4. **Maintain seam-gated observation** — Sentinels observe transitions between layers
   without owning truth
5. **Freeze doctrine before building** — The Startup Freeze Pack establishes immutable
   doctrine before any runtime code enters the system

## Architecture Principles

- Doctrine above code
- Governance above domain
- Truth above convenience
- Lineage above absorption
- Observation above inference
- Determinism above heuristics

## Non-Goals

- Construction OS is not a general-purpose OS
- Construction OS does not replace ValidKernel governance
- Construction OS does not own Universal_Truth_Kernel
- Construction OS does not absorb sovereign fabric-level repos

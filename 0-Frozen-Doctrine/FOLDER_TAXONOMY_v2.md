# Folder Taxonomy v2

## Authority
Armand Lefebvre, L0 — Lefebvre Design Solutions LLC

## Classification
FROZEN DOCTRINE — Ring 0

## Purpose
Defines the canonical folder hierarchy of the Construction OS v2 monorepo.

## Taxonomy

### Ring 0 — 0-Frozen-Doctrine
Frozen governance doctrine. Immutable after genesis except by L0 command.
Contains: constitution, governance package, architect intent, engineering matrix,
consolidation doctrine, boundary doctrine, signal doctrine, sentinel doctrine,
folder taxonomy, doctrine lock, truth model, and all sentinel/governance specs.

### Ring 1 — 1-Governance-Registry-Runtime
Governance authority, registry topology, and runtime execution surfaces.
Contains absorbed governance contracts, registry entries, and runtime configurations.

### Ring 2 — 2-Engines-Tools-Datasets
Engine definitions, tool registries, and dataset schemas.
Contains the computational and data surfaces of the Construction domain.

### Ring 3 — 3-Applications-Control-Towers
Application-layer definitions and control tower configurations.
Contains user-facing orchestration surfaces.

### Ring 4 — 4-Workers-Relays-Assistants
Worker definitions, relay configurations, and assistant interfaces.
Contains execution and communication surfaces.

### Ring 5 — 5-State-Receipts-Signals
State snapshots, governance receipts, and signal definitions.
Contains observability and audit surfaces.

### Ring 6 — 6-Archive-Lineage
Absorption receipts, lineage manifests, and archive records.
Contains the permanent record of all consolidation actions.

## Frozen State
This taxonomy is frozen as of Construction OS v2 genesis.
Modification requires L0 command authority.

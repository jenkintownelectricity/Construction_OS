# Boundary: 5-State-Receipts-Signals

## Classification
Ring 5 — State, Receipts, and Signals

## Purpose
Contains state snapshots, governance receipts, and signal definitions.
These are the observability and audit surfaces of the Construction OS domain.

## Rules
- No code files (.py, .js, .ts, .tsx, .rs) in this folder during Wave 0
- State files must be JSON or Markdown only
- Receipts must conform to ValidKernel-Governance receipt schemas
- Signals must align with Construction_Cognitive_Bus signal contracts
- No content may override any higher-ring doctrine

## Boundary Lock
This folder boundary is frozen as of Construction OS v2 genesis.
Modification requires L0 command authority.

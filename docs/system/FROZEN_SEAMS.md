# FROZEN_SEAMS — Construction_Kernel

## Frozen Definitions

These elements are structurally fixed and must not be casually redefined.

1. **Governance Kernel** — Owns governance truth for construction domain.
2. **Geometry Kernel** — Owns geometric truth for construction domain.
3. **Chemistry Kernel** — Owns material/chemistry truth for construction domain.
4. **Assembly Kernel** — Owns assembly truth for construction domain.
5. **Reality Kernel** — Owns site-reality truth for construction domain.
6. **Deliverable Kernel** — Owns deliverable truth for construction domain.
7. **Intelligence Kernel** — Owns intelligence/reasoning truth for construction domain.

### Kernel Ownership Boundaries

Each kernel has exclusive ownership of its domain truth. Kernels consult but do not replace each other.

### Kernel Relationship Flow

- Governance + Geometry + Chemistry → Assembly
- Assembly + Reality → Deliverables
- All kernels → Intelligence

### Nucleus Inheritance

Nucleus truth inheritance — this kernel operates within the truth boundary defined by Universal_Truth_Kernel. This inheritance is frozen.

## Boundary Rules

- Construction Kernel defines applied construction truth; does not originate universal truth.
- Each kernel has exclusive ownership of its domain.
- Kernels consult but do not replace each other.

## What May Evolve

- Kernel internal definitions and properties
- New kernel properties (additive, not identity-breaking)
- App spec details
- Relationship map details (refinement, not restructuring)

## Invalid Drift

The following constitute structural violations:

- Construction Kernel claiming universal truth origination
- Kernel ownership overlap (two kernels owning the same truth)
- Runtime redefining kernel truth
- App layer redefining kernel boundaries

## Safe Changes

- Extending kernel definitions within existing boundaries
- Adding new properties to kernels
- Refining app specs
- Updating maps to reflect new detail

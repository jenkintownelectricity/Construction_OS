# Stack Position

## Layer Position

Construction_Assistant does not occupy a numbered layer. It sits beside the stack, adjacent to Layers 5-7.

## Consumed Layers

| Layer | System | What the Assistant Consumes |
|---|---|---|
| Layer 5 | Construction_Kernel | Domain truth from 7 kernels (Governance, Geometry, Chemistry, Assembly, Reality, Deliverable, Intelligence) |
| Layer 6 | Construction_Runtime | Execution state, pipeline positions, validation results from parse/normalize/validate/generate/audit pipeline |
| Layer 7 | Construction_Application_OS | Application state from Assembly Parser and Spec Intelligence |

## Referenced Layer

| Layer | System | Relationship |
|---|---|---|
| Layer 0 | Universal_Truth_Kernel | Conceptual reference for doctrinal alignment. Not directly consumed. The assistant does not read Layer 0 artifacts at runtime. |

## Position Constraints

1. The assistant is not between any two layers. It is beside the stack.
2. The assistant does not intercept data flowing between layers.
3. The assistant does not gate or filter data flowing between layers.
4. The assistant reads truth surfaces that layers expose. It does not access internal layer state.
5. The assistant has no write path to any layer.

## Why Beside, Not Inside

The assistant is a consumer of governed truth, not a producer or processor of it. Placing it inside the stack would imply it participates in truth production, validation, or execution. It does none of these. Its position beside the stack reflects its role as a bounded, read-only, emission-only interface.

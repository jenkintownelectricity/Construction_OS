# Future Expansion Notes

## Status

All items below are NOT YET BUILT. This document records bounded expansion directions that remain consistent with assistant doctrine. No item here implies commitment, timeline, or approval.

## Potential Expansion Areas

### Voice Interface

- Voice-based operator queries routed through the same truth retrieval and emission classification pipeline.
- All doctrine constraints (no mutation, no truth origination, bounded emission) apply identically.
- Voice responses must still carry emission class metadata.
- Not yet built. No design work started.

### Mobile Interface

- Mobile-optimized query interface for field operators.
- Same bounded emission model applies. No special authority granted by interface form factor.
- Must preserve emission classification visibility on constrained screen sizes.
- Not yet built. No design work started.

### Workflow Visualization

- Read-only visualization of workflow state retrieved from Construction_Application_OS (Layer 7).
- The assistant would render governed state, not maintain its own workflow model.
- Visualization does not constitute execution or approval.
- Not yet built. No design work started.

## Expansion Constraints

1. No expansion may grant the assistant truth origination authority.
2. No expansion may grant the assistant state mutation capability.
3. No expansion may bypass emission classification.
4. No expansion may alter the assistant's position (beside the stack, not inside it).
5. All expansions must be governed by the same doctrine and frozen seams defined in this repo.
